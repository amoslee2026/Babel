# IFP 安装配置指南

本文档详细说明IC Flow Platform的安装过程、系统配置、环境设置和故障排除。

---

## 1. 系统要求

### 1.1 硬件要求

| 项目 | 最低要求 | 推荐配置 |
|------|----------|----------|
| CPU | 2核 | 4核+ |
| 内存 | 4GB | 8GB+ |
| 磁盘 | 500MB（IFP本身） | 根据项目数据确定 |
| 显示 | 1024×768 | 1920×1080+ |

### 1.2 软件要求

| 项目 | 版本要求 | 说明 |
|------|----------|------|
| OS | Linux CentOS/RHEL 7+ | 或兼容发行版 |
| Python | 3.8+ | 推荐3.10，建议使用Anaconda/Miniconda |
| Qt | 5.15+ | PyQt5依赖 |
| Graphviz | 2.40+ | 流程图生成 |

### 1.3 网络要求

| 场景 | 要求 |
|------|------|
| GitHub下载 | 可访问github.com或使用镜像 |
| PyPI下载 | 可访问pypi.org或使用国内镜像 |
| LSF集群 | 需配置LSF环境（可选） |

**install.py源码解析**:

```python
# install.py 关键逻辑

import os
import sys
import subprocess

def check_python_version():
    """检查Python版本"""
    if sys.version_info < (3, 8):
        print("ERROR: Python 3.8+ required")
        sys.exit(1)
    print(f"Python version: {sys.version}")

def check_dependencies():
    """检查依赖"""
    required = [
        'PyQt5', 'yaml', 'graphviz', 'pandas',
        'numpy', 'psutil', 'matplotlib', 'xlrd', 'xlwt'
    ]
    for pkg in required:
        try:
            __import__(pkg)
            print(f"  {pkg}: OK")
        except ImportError:
            print(f"  {pkg}: MISSING")
            return False
    return True

def create_wrapper(install_path, python_path):
    """创建wrapper脚本"""
    wrapper_content = f'''#!/bin/bash
# IFP Wrapper Script
# Generated by install.py

# Python路径
export PYTHON_PATH="{python_path}"

# IFP安装路径
export IFP_INSTALL_PATH="{install_path}"

# 库路径（某些系统需要）
export LD_LIBRARY_PATH="{install_path}/lib:$LD_LIBRARY_PATH"

# 添加到Python路径
export PYTHONPATH="{install_path}/bin:{install_path}/common:{install_path}/config:$PYTHONPATH"

# 执行IFP
exec {python_path} {install_path}/bin/ifp.py "$@"
'''
    
    wrapper_path = os.path.join(install_path, 'bin', 'ifp_wrapper.sh')
    with open(wrapper_path, 'w') as f:
        f.write(wrapper_content)
    
    os.chmod(wrapper_path, 0o755)
    
    # 创建符号链接
    link_path = os.path.expanduser('~/bin/ifp')
    if os.path.exists(os.path.expanduser('~/bin')):
        os.symlink(wrapper_path, link_path)
    
    return wrapper_path

def main():
    install_path = os.path.dirname(os.path.abspath(__file__))
    python_path = sys.executable
    
    print("=" * 50)
    print("IFP Installation")
    print("=" * 50)
    
    # 1. 检查Python版本
    check_python_version()
    
    # 2. 检查依赖
    print("\nChecking dependencies...")
    if not check_dependencies():
        print("\nPlease install missing dependencies:")
        print("  pip install -r requirements.txt")
        sys.exit(1)
    
    # 3. 创建wrapper
    print("\nCreating wrapper script...")
    wrapper_path = create_wrapper(install_path, python_path)
    print(f"  Wrapper created: {wrapper_path}")
    
    # 4. 设置环境变量提示
    print("\n" + "=" * 50)
    print("Installation Complete!")
    print("=" * 50)
    print("\nAdd to your ~/.bashrc:")
    print(f'  export IFP_INSTALL_PATH="{install_path}"')
    print(f'  export PATH="$HOME/bin:$PATH"')
    print("\nThen:")
    print("  source ~/.bashrc")
    print("  ifp")

if __name__ == '__main__':
    main()
```

## 6. 常见问题与解决方案

### 6.1 PyQt5安装问题

```bash
# 问题: pip安装PyQt5失败
# ERROR: Could not find a version that satisfies the requirement PyQt5

# 解决方案1: 使用conda
conda install pyqt

# 解决方案2: 使用系统包
sudo dnf install python3-qt5

# 解决方案3: 指定版本
pip install PyQt5==5.15.9 --no-cache-dir

# 问题: Qt platform plugin错误
# "could not find the Qt platform plugin 'xcb' in ''

# 解决方案: 安装X11依赖
sudo dnf install \
    xcb-util-renderutil \
    xcb-util-image \
    xcb-util-keysyms \
    xcb-util-wm \
    libxkbcommon-x11

# 设置环境变量
export QT_QPA_PLATFORM=xcb
export QT_DEBUG_PLUGINS=1  # 调试输出
```

### 6.2 Graphviz问题

```bash
# 问题: graphviz Python包安装成功但找不到dot命令
# FileNotFoundError: [Errno 2] No such file or directory: 'dot'

# 解决方案: 安装系统graphviz
sudo dnf install graphviz

# 验证
which dot
# /usr/bin/dot

dot -V
# dot - graphviz version 2.40.1

# 问题: 流程图生成失败
# Error: failed to execute 'dot'

# 解决方案: 检查PATH
export PATH="/usr/bin:$PATH"

# 或指定graphviz路径
python3 -c "
import graphviz
graphviz.DOT_PATH = '/usr/bin/dot'
"
```

### 6.3 权限问题

```bash
# 问题: 无法创建ifp.cfg.yaml
# PermissionError: [Errno 13] Permission denied: 'ifp.cfg.yaml'

# 解决方案1: 使用只读模式
ifp -r

# 解决方案2: 检查目录权限
ls -la .
# 检查是否有写权限

chmod u+w .
# 或使用有权限的目录
cd ~/project
ifp

# 问题: SQLite数据库锁定
# sqlite3.OperationalError: database is locked

# 解决方案: 清理缓存
rm -rf .ifp.cache/db/*.db-journal

# 或使用独立数据库
export IFP_DB_PATH=/tmp/ifp_status.db
```

### 6.4 LSF连接问题

```bash
# 问题: bsub命令找不到
# NameError: name 'bsub' is not defined

# 解决方案: 配置LSF环境
source /opt/lsf/conf/profile.lsf

# 或设置PATH
export PATH="/opt/lsf/bin:$PATH"

# 问题: LSF提交失败
# LSF submission failed: Cannot connect to LIM

# 解决方案1: 检查LSF服务
lsadmin limstat
badmin mstat

# 解决方案2: 检查配置
cat /opt/lsf/conf/lsf.conf
# 确认LSF_MASTER配置正确

# 问题: LSF监控无法启动
# Flask server failed to start

# 解决方案: 检查端口占用
netstat -tlnp | grep 5000
# 如有占用，换端口
python3 monitor/app.py --port 5001
```

## 9. 生产环境部署建议

### 9.1 安装路径规范

```
# 推荐结构
/opt/ifp/                  # IFP安装路径
├── current/               # 当前版本（软链接）
│   └── bin/ifp.py
├── v1.4.3/                # 具体版本
├── v1.4.2/                # 旧版本（备份）
├── config/                # 全局配置
│   ├── default.yaml
│   └── api.yaml
└── logs/                  # 系统日志
```

### 9.2 systemd服务配置

```bash
# 创建服务文件
sudo vim /etc/systemd/system/ifp-monitor.service

[Unit]
Description=IFP LSF Monitor Service
After=network.target

[Service]
Type=simple
User=ifpadmin
Group=ifp
WorkingDirectory=/opt/ifp/current/tools/lsfMonitor
Environment="IFP_INSTALL_PATH=/opt/ifp/current"
ExecStart=/opt/ifp/current/tools/lsfMonitor/monitor/app.py --port 5000
Restart=on-failure
RestartSec=10

[Install]
WantedBy=multi-user.target

# 启用服务
sudo systemctl daemon-reload
sudo systemctl enable ifp-monitor
sudo systemctl start ifp-monitor

# 查看状态
sudo systemctl status ifp-monitor
```

### 9.3 多用户配置

